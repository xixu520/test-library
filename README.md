# PDF 文档管理与 OCR 核验系统

这是一个基于 Go、Python 和 React 栈构建的企业级 PDF 标准文档管理系统。提供文档上传、PDF 分类存储、异步高精度 OCR 识别、文档废止状态核验、以及动态系统配置（例如阿里云高速 OCR 接入）等全流程自动化能力。

## 系统架构与技术栈

整个平台采用微服务架构（Microservices），各职责模块分离，通过提供标准化 API 相互通信：

*   **前端（Web UI）**: React + Vite，提供响应式的用户界面。
*   **网关与主要业务端（API-Go）**: Go + Fiber + GORM。负责安全认证、权限控制、文档的增删改查、以及 PostgreSQL 的写入。
*   **光学字符识别与挖掘端（OCR-Python）**: FastAPI + pdfplumber + Tesseract。负责无头异步处理、清洗并抽取出大体积扫描 PDF 中的标准号和各个有效期限。
*   **数据库（Database）**: PostgreSQL 16。用于固化文件索引和全局配置。

## 部署先决条件

*   系统必须安装有支持容器化运行环境的引擎，例如 **Docker** (>= 20.10) 和 **Docker Compose** (>= v2)。
*   服务器需预留并开放相应的网络端口：`3000` (Web端), `8080` (Go服务端), `8001` (Python服务端), `5432` (PostgreSQL数据库)。
*   *(重要)* 为了最佳体验，主机需要能够连接到外网（比如阿里云）以实时完成标准生命周期的远程核验以及高级 OCR 的智能纠错调用。

## 快速部署指南 (Docker 方式)

项目已为您配置好了开箱即用的 `docker-compose.yml` 脚本。建议使用独立网桥通过容器直连，避免复杂的外部依赖。

1. **克隆并进入项目根目录：**
   将所有微服务源码以及 `docker-compose.yml` 存放于同一个目录结构中。

2. **一键构建并启动所有服务集群：**
   ```bash
   docker compose up -d --build
   ```
   > 提示：首次构建会安装 Python 依赖库和 Go 远端包，依据您的外网质量可能需要几分钟的时间。

3. **验证服务健康状态：**
   启动完成后，您可以使用终端命令查看四个主要容器是否处于 `healthy` 或者 `Up` 的运行态。
   ```bash
   docker compose ps
   ```

4. **访问平台：**
   *   打开您的现代浏览器设备，访问 **`http://localhost:3000`** （若是外网服务器请替换为公网 IP）。
   *   初次运行系统会执行初始化挂载脚本，自动创建一个拥有最高权限的超级管理员账户：
       *   **初始登录账号：** `admin`
       *   **初始登录密码：** `admin123`
   *   您可前往平台右上角进行密码更改与配置其他子用户。

## 进阶全局配置与灾备

### 环境变量与机密管理
生产环境下，请务必修改根目录中的 `docker-compose.yml` 及其机密信息：
- 更改 `api-go` 服务中的 `JWT_SECRET` 为至少 32 位的强随机字符串以保障 Session 不被反解。
- 更改 `postgres` 根密码 `POSTGRES_PASSWORD` 与 `api-go` 中的配置保持一致。

### 阿里云远程 OCR 配置
系统在遇见部分受损或深度扫描件时可能会进行本地 Tesseract 的兜底处理导致效率受限。强烈推荐使用平台自带的阿里云拓展功能：
1. 登录系统后，点击右上角管理员用户名选择 **“系统设置”**。
2. 将您事先获取的 **阿里云 Access Key ID** 与 **Access Key Secret** 输入后点击保存应用。
3. 返回文档列表，在报错或是缺失字段的行级操作处，点击新出现的 **🌐远程OCR**，系统将单独切割此 PDF 首页传入高精度处理网关校验并回写数据。

### 文件挂载与存储持久化
默认情况下，数据库内容与 PDF 的真实持久化载体是通过 Docker Volume 挂载至宿主机的：
- 数据库卷材名称：`pg_data`
- 文档落盘挂载名称：`pdf_storage`

备份时可直接将对应的 Volume 进行打包导出，即可完成全量数据的跨云/跨机灾备转移。

## 开发与调试 (独立环境)

如欲二次开发对应组件，可直接进入子目录，根据其中的环境提示进行本地运行。
例如，本地开发 `web-ui` 端：
```bash
cd web-ui
npm install
npm run dev
```

本地调优 `ocr-python` 端：
```bash
cd ocr-python
pip install -r requirements.txt
python -m uvicorn main:app --reload --port 8001
```
