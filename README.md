# PDF 文档管理与 OCR 核验系统

这是一个基于 Go、Python 和 React 栈构建的企业级 PDF 标准文档管理系统。提供文档上传、PDF 分类存储、异步高精度 OCR 识别、文档废止状态核验、以及动态系统配置（例如阿里云高速 OCR 接入与实时日志监控）等全流程自动化能力。

## 系统架构与技术栈

整个平台采用微服务架构（Microservices），各职责模块分离，通过提供标准化 API 相互通信：

*   **前端（Web UI）**: React + Vite，提供响应式的用户界面，支持实时配置热更新。
*   **网关与主要业务端（API-Go）**: Go + Fiber + GORM。负责安全认证、高级别权限控制、文档增删改查、以及 PostgreSQL 的并发事务写入。
*   **光学字符识别与挖掘端（OCR-Python）**: FastAPI + pdfplumber + Tesseract。内置内存级日志截获器与高速流式解析算子。提供本地 Tesseract 免费算力与阿里云高级通用文字识别的双引擎混合驱动。
*   **数据库（Database）**: PostgreSQL 16。用于固化文件索引并提供高性能的 ILIKE 全文检索。

## 🌟 核心增值特性

本系统除了基础的 CRUD 操作外，更着重于解决真正的文档归档痛点：

- **智能摘要识别引擎 (OCR):** 依据上传文档的第一页，系统自动截取和识别标准编号，避免无效的全篇扫描导致资源浪费。
- **自动推断与校正:** 自动解析诸如 `GB` (国家标准), `JC/JG/JGJ/CJJ` (行业标准), `DB` (地方标准), `T` (团体标准) 的首字母规则，全自动归类标准类型。
- **自由干预与无缝重新触发:** 所有机器识别的标准号、发布/实施日期都允许**手动修改与无缝介入**，并支持“重新识别 OCR” 和 “重新核验”。
- **灵活的回收与灾备方案:** 误删的文档会被送入专用的“回收站”。支持一键恢复还原至原目录、强制销毁与清空回收站机制，保证误操冗余度与数据隐私。

## 部署先决条件

*   系统必须安装有支持容器化运行环境的引擎，例如 **Docker** (>= 20.10) 和 **Docker Compose** (>= v2)。
*   服务器需预留并开放相应的网络端口：`3000` (Web端), `8080` (Go服务端), `8001` (Python服务端), `5432` (PostgreSQL数据库)。
*   *(重要)* 为了最佳体验，主机需要能够连接到外网（比如阿里云）以实时完成标准生命周期的远程核验以及高级 OCR 的智能纠错调用。

## 快速部署指南 (Docker 方式)

项目已为您配置好了开箱即用的 `docker-compose.yml` 脚本。建议使用独立网桥通过容器直连，避免复杂的外部依赖。

1. **克隆并进入项目根目录：**
   将所有微服务源码以及 `docker-compose.yml` 存放于同一个目录结构中。

2. **一键构建并启动所有服务集群：**
   ```bash
   docker compose up -d --build
   ```
   > 提示：首次构建会安装 Python 依赖库和 Go 远端包，依据您的外网质量可能需要几分钟的时间。如果需要清除缓存重新构建，可以使用 `--no-cache`。

3. **访问平台：**
   *   打开您的现代浏览器设备，访问 **`http://localhost:3000`** （若是外网服务器请替换为公网 IP）。
   *   初次运行系统会执行初始化挂载脚本，自动创建一个拥有最高权限的超级管理员账户：
       *   **初始登录账号：** `admin`
       *   **初始登录密码：** `admin123`
   *   您可前往平台右上角进行密码更改与配置其他子用户。

## 进阶使用与监控 (管理员专座)

### 阿里云远程 OCR 配置与连通性测试
系统在遇见部分受损或深度扫描件时可能会进行本地 Tesseract 的兜底处理导致效率受限。强烈推荐使用平台自带的阿里云拓展功能：
1. 登录系统后，点击右上角管理员控制台的 **“⚙️ 系统设置”**。
2. 将您事先获取的 **阿里云 Access Key ID** 与 **Access Key Secret** 输入。
3. 点击 **🔗 测试 API 连通性**：系统会在后台自动生成一张极小的单像素图片流打入您的认证网关。如果无误则会立刻呈现 `✅ 连通正常` 绿色标识。若存在尾随空格，或账户余额不足，报错也将以红字直观展露在前端！
4. 返回文档列表，在需要重试或空白的行，点击 **🌐远程OCR**，系统将调用高速商用引擎进行高精度提取并修正。

### 实时日志透视窗
抛弃传统的服务端终端 `tail -f`：在系统设置面板的下方，内置了**实时日志抓取台**。
系统会高频自动侦测远端（Python OCR）容器中打印的所有日志流并在浏览器前端进行滚动推送。您可以轻松观测您的识别进度、系统报错及拦截情况，真正做到对微服务状态心中有数！

### 文件挂载与存储持久化
默认情况下，数据库内容与 PDF 的真实持久化载体是通过 Docker Volume 挂载至宿主机的：
- 数据库卷材名称：`pg_data`
- 文档落盘挂载名称：`pdf_storage`

备份时可直接将对应的 Volume 进行打包导出，即可完成全量数据的跨云/跨机灾备转移。

## 开发与调试 (独立环境)

如欲二次开发对应组件，可直接进入子目录，根据其中的环境提示进行本地运行。
例如，本地开发 `web-ui` 端：
```bash
cd web-ui
npm install
npm run dev
```

本地调优 `ocr-python` 端：
```bash
cd ocr-python
pip install -r requirements.txt
python -m uvicorn main:app --reload --port 8001
```
